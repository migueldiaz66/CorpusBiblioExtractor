# Info & References — Supported Files (bases y formatos soportados)

> Metadata de biblioshiny 5.4.1, extraída de `inst/biblioshiny/helpContent.R` (clave `info`).
> Ruta de menú: Info › Supported Files · tabName: `supFiles`

## Resumen (español)

- Esta página lista las **bases bibliográficas** que Biblioshiny puede importar y analizar: Web of Science, Scopus, OpenAlex, Dimensions, The Lens, PubMed y Cochrane Library.
- **Web of Science, Scopus y OpenAlex** permiten exportar el conjunto completo de metadatos, habilitando todos los análisis de Biblioshiny.
- **Dimensions, PubMed y Cochrane Library** ofrecen metadatos limitados, lo que restringe los análisis posibles.
- Incluye una tabla con, por cada base: formatos de exportación soportados, tipos de metadatos y el **formato recomendado** (p. ej. Plaintext para WoS, CSV para Scopus, API para OpenAlex, Excel para Dimensions).
- Nota: la **API de Dimensions ya no está soportada** (paso a un modelo de acceso comercial cerrado).
- Sección especial sobre **importación desde OpenAlex**: vía exportación web CSV/Excel (con selección de columnas) o vía **API** (metadatos completos, incluidas referencias citadas).
- Campos **obligatorios** del export web de OpenAlex: Work ID, Title, Author, Source, Year; sin ellos no se puede crear la colección. Se recomiendan campos adicionales para habilitar más análisis.
- La exportación web de OpenAlex **no incluye referencias citadas**, por lo que los análisis basados en citas requieren la API o la opción "Fetch/Resolve cited references".

## Info & References (original, en inglés)

### 📚 Supported Bibliographic Databases and Suggested File Formats

**Biblioshiny** imports and analyzes collections exported from the following bibliographic databases:

- [Web of Science](https://www.webofscience.com)
- [Scopus](https://www.scopus.com)
- [OpenAlex](https://www.openalex.org)
- [Dimensions](https://www.dimensions.ai)
- [The Lens](https://www.lens.org)
- [PubMed](https://pubmed.ncbi.nlm.nih.gov)
- [Cochrane Library](https://www.cochranelibrary.com)

**Web of Science**, **Scopus**, and **OpenAlex** allow users to export the complete set of metadata, making it possible to perform all analyses implemented in **Biblioshiny**.

Some other databases, such as **Dimensions**, **PubMed**, and **Cochrane Library**, provide only a limited set of metadata. This may impose restrictions on the range of analyses that can be conducted using those datasets.

The following table reports, for each supported database:

- The **file formats supported** by the export interface
- The **types of metadata** contained in each export option
- The **suggested file format** to use with **Biblioshiny**

#### Databases, available metadata and suggested file formats

| Source | Format | Exported metadata | Recommended Format |
| --- | --- | --- | --- |
| *Web of Science* | 'BibTeX'<br>'Plaintext'<br>'EndNote Desktop' | All except Cited references<br>All<br>All | **Plaintext** |
| *Scopus* | 'BibTeX'<br>'CSV' | All except Cited references<br>All | **CSV** |
| *OpenAlex* | 'CSV / Excel' (web export)<br>'API' | User-selected columns; **no** cited references (see OpenAlex section below)<br>All metadata, including cited references | **API** |
| *Dimensions* | 'Excel'<br>'API' (no longer supported — Dimensions has moved to a closed commercial access model) | All but with limited reference info (only Ids)<br>— | **Excel** |
| *The Lens* | 'CSV export' | All but with limited reference info (only Ids) | **CSV** |
| *PubMed* | 'PubMed export'<br>'API' | Document, Authors, Content info<br>Document, Authors, Content info | **PubMed export** |
| *Cochrane Library* | 'Plaintext' | Document, Authors, Content info | **Plaintext** |

### 🅾️ Importing from OpenAlex

**OpenAlex** can be imported into Biblioshiny in two ways:

- **CSV / Excel web export** — downloaded from the OpenAlex website using the *Export* button (`Works > Export`). When exporting, OpenAlex lets you **choose which columns (metadata fields) to include**. The set of available analyses in Biblioshiny depends directly on the columns you select.
- **OpenAlex API** — retrieved programmatically (via the `openalexR` package or the API import option). This returns the **full metadata**, including the list of **cited references** needed for citation-based analyses.

#### ⚠️ Which columns to select in the web export?

When you export a CSV/Excel file from the OpenAlex website, make sure to include **at least the required fields**. If any of them is missing, Biblioshiny cannot build the bibliographic collection and the import is stopped with an explanatory message.

| Level | OpenAlex export columns | Why they are needed |
| --- | --- | --- |
| **Required** (minimum set) | **Work ID**, **Title**, **Author**, **Source**, **Year** | Unique identifier (de-duplication, short reference), document, author and source identification, and publication year. Without these the collection cannot be created. |
| **Recommended** | **Citation count**, **DOI**, **Author IDs**, **ORCID**, **Institution** & **Institution IDs**, **Country**, **Language**, **Abstract**, **Corresponding author** & **Corresponding institution**, **Keywords / Concepts** | Enable impact analysis, author disambiguation, affiliation and country analysis, language filtering, text/content analysis (abstract, keywords) and corresponding-author analysis. The import still works without them, but the related analyses are limited or unavailable. A warning lists the missing fields. |

#### 🔗 Cited references: enrich your collection via the API

The OpenAlex **web CSV/Excel export does not include the cited references** (and does not include the document type). Therefore **citation-based analyses** — such as *Historical Direct Citation Network*, *Co-citation* and *Bibliographic Coupling* — are **not available** from a plain web export.

To obtain them you can:

- Import the collection through the **OpenAlex API** (recommended for complete metadata), or
- After importing a CSV/Excel file, use the **"Fetch/Resolve cited references"** option, which downloads the missing reference data for your documents directly from the **OpenAlex API**.

*Note:* reference enrichment via the API requires an internet connection and may take some time for large collections.
